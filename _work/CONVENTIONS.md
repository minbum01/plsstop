# 추출 규칙 (합격선 관리 v2 양식)

각 행은 **23개 필드 배열**(아래 순서 0~22)로 출력한다. 빈 값은 빈 문자열 "".

| idx | 컬럼 | 설명 |
|---|---|---|
| 0 | 지역 | 예: 서울, 경기, 강원 (PDF가 속한 지역 폴더명) |
| 1 | 시험종류 | `공무원`(9급/7급 일반직) 또는 `교행`(교육청) |
| 2 | 연도 | `2026` 등 |
| 3 | 회차 | `1회`,`2회`,`3회`… (제목의 "제N회"에서). 없으면 "" |
| 4 | 시험구분 | `공개경쟁` 또는 `경력경쟁` |
| 5 | 임용예정기관 | 시·군·구/의회/교육지원청명. 단일기관(서울/부산 등)이면 "" |
| 6 | 직군 | `행정직군`/`기술직군`. (규칙 아래) |
| 7 | 직렬 | 예: 행정, 세무, 공업, 교육행정 |
| 8 | 직류 | 예: 일반행정, 지방세, 일반기계, 교육행정 |
| 9 | 직급 | `9급`/`8급`/`7급` |
| 10 | 대상 | `일반`(기본)/`장애인`/`저소득층`/`지방의회`/`보훈청추천` 등 |
| 11 | 선발예정인원 | 정수 (콤마 제거) |
| 12 | 접수인원 | 정수. 남/여/합계로 나뉘면 **합계** 사용 |
| 13 | 경쟁률(접수/선발) | `"5.4:1"` 형식 (공백 제거). "-"면 "" |
| 14 | 응시인원 | 접수현황만 있으면 "" |
| 15 | 응시율(%) | "" |
| 16 | 경쟁률(응시/선발) | "" |
| 17 | 필기합격인원 | "" |
| 18 | 합격선 | "" |
| 19 | 합격선기준 | "" |
| 20 | 양성평등합격선 | "" |
| 21 | 원본파일명 | PDF 실제 파일명(확장자 .pdf 포함) |
| 22 | 비고 | `미선발` 등 특이사항. 없으면 "" |

## 공통 규칙 (2026 접수현황: 9급·교행)
- 접수현황은 **선발/접수/경쟁률(접수/선발)만** 채우고 14~20은 전부 "".
- **소계·총계·직군소계·합계 행은 출력하지 않는다.** 말단(leaf) 행만 출력.
  - 시·군별로 펼쳐진 PDF: 각 시·군(및 경기도 본청, ○○시의회 등)을 **임용예정기관**으로 한 행씩. 소계는 버림.
  - 시·군 구분이 없는 단일기관(서울/부산/대구/광주/대전/울산/인천 등): 임용예정기관="" 로 직렬/직류/대상별 한 행씩.
- **회차**: 제목의 "제N회" → "N회". 한 PDF에 공개경쟁/경력경쟁 회차가 다르면(예: 충북 공개1회+경력2회) 각 행의 시험구분에 맞는 회차를 쓴다.
- **접수인원/경쟁률**: 콤마 제거. 경쟁률 "5.4 : 1"→"5.4:1". 접수 "-"(미응모)면 접수=0, 경쟁률="".
- **선발 0 / 접수 0 (미선발)**: 비고에 "미선발" 기재.

## 직렬/직류/대상 파싱
양식이 PDF마다 다르다. 세 가지 패턴:
1. **분리형(서울)**: 직렬 열 + 직류(분야) 열이 따로. 그대로 사용. 직류에 붙은 (장애인)/(저소득층)/(지방의회) 등은 떼서 **대상**으로.
   - 예: 직렬=행정, 직류=`일반행정(지방의회)` → 직류=일반행정, 대상=지방의회.
   - 예: 직류=`의료기술(방사선)` → 직류=의료기술, 대상=방사선.
2. **결합형(경기)**: `행정9급(일반행정)` = 직렬+직급+(직류). → 직렬=행정, 직급=9급, 직류=일반행정.
   - `행정9급(일반행정)(장애인)` → 직렬=행정, 직급=9급, 직류=일반행정, 대상=장애인.
3. **교행형(강원)**: `직렬(직류)` 열 = `교육행정(교육행정)`,`공업(일반기계)`,`시설(건축)`. → 직렬=교육행정/공업/시설, 직류=교육행정/일반기계/건축. 대상은 같은 직렬 아래 일반/장애인/저소득층 하위행에서.
- 대상 기본값 = `일반`. (장애인)/(저소득층)/(지방의회)/(보훈청추천)/(방사선)/(물리치료) 등이 붙으면 그 값.

## 직군(idx6) 규칙
- PDF에 직군 헤더가 **명시되면 그대로** 사용(서울: 행정직군/과학기술직군 → "과학기술직군"은 그대로 둠).
- 명시 안 됐으면 **직렬로 추정**:
  - 행정직군: 행정, 세무, 전산, 사회복지, 사서, 속기, 방호, 감사, 통계, 교정, 보호, 검찰, 출입국관리, 직업상담, 운전, 위생, 사회복지
  - 기술직군: 공업, 농업, 녹지, 시설, 보건, 간호, 환경, 방재안전, 방송통신, 시설관리, 해양수산, 의료기술, 보건진료, 수의, 축산, 식품위생, 조경, 데이터
  - 교행(시험종류=교행)은 직군을 **빈칸 ""** 으로 둔다(v2 교행은 직군 비어있음).
  - 위 목록에 없는 직렬은 추정하지 말고 직군="" + flags에 기록.

## 출력
파일을 JSON으로 작성: `{"region":"서울","category":"9급접수","source":"<파일명>","rows":[[...23...],...],"checks":{"sum_선발":N,"sum_접수":N,"doc_총계_선발":N,"doc_총계_접수":N},"flags":["..."]}`
- `checks`: 본인이 출력한 모든 행의 선발/접수 합계와, 문서 상단 총계(총계/합계) 값을 같이 적어 검증 가능하게.
- `flags`: 애매했던 점, 읽기 어려운 셀, "-" 처리, 직군 추정 불가, 회차 불명확 등 **추측하지 말고 전부 기록**.
