## 0. 목표
`합격선_관리_v2.html`(자체완결형 데이터 뷰어, 23필드 배열 9,380행)에
**① 지방직 9급 2026 접수현황 ② 교행 2026 접수현황 ③ 7급 응시율·합격선**을 추가해
`합격선_관리_v3.html`로 만든다.

## 1. 작업 방식 (중요)
- **자율/병렬 에이전트로 PDF 막 추출 금지.** (지난번 실패 경험)
- 기본: **사용자가 PDF 텍스트를 복붙 → Claude가 v2 23칸 양식 행으로 변환.**
- 복붙이 깨지는 PDF(병합셀로 기관·라벨이 분리됨)는 **Claude가 비전(Read)으로 직접 판독.**
- **모든 지역은 "소계=시·군 합" + "문서 총계 일치" 산술검산을 통과시킨 뒤에만 반영.**
- 애매한 값(특히 공개경쟁/경력경쟁 시험구분)은 **추측하지 말고 빈칸**으로 두되 데이터 행은 유지.

## 2. 데이터 구조 (v2/v3 공통)
- `합격선_관리_v2.html` 안의 한 줄짜리 거대 배열:
  - `const RAW=[[...23필드...],...]` (line 67) — 실제 데이터
  - `const DONE=[...],TOTAL=250,LINKS={...},ISSUES=[...]` (line 69)
- 23필드 순서/규칙: **`_work/CONVENTIONS.md` 참조** (직군 추정표, 직렬/직류/대상 파싱규칙 등)
- 핵심: 시·군/교육지원청별로 행 분리(임용예정기관 칸), 소계/총계 행은 제외, 7급 직군은 직렬로 추정, 합격선기준은 원본 표기 그대로.

## 3. v3 빌드 방법 (재실행 가능)
```
PYTHONUTF8=1 python _work/build_v3.py
```
- `_work/out/*.json`(지역별 추출결과)을 전부 모아 v2의 RAW에 추가 → `합격선_관리_v3.html` 재생성.
- **새 지역을 추가하면 그 지역 json을 `_work/out/`에 만들고 위 명령 다시 실행하면 됨.**
- 현재 v3 전체: **12,033행** (기존 9,380 + 신규 2,653)

- 입력 복붙: `9급 2026 접수/응시현황.md` (지역 텍스트를 이어붙이는 작업파일)
